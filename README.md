# The Herbert Team — Mortgage Website

Production marketing site for **The Herbert Team**, the mortgage practice of **Jason Herbert** (NMLS #633039), a licensed mortgage advisor originating through **Primis Mortgage Company** (NMLS #1894879) from Georgetown, Texas.

- **Positioning:** The Mortgage Solution Expert
- **Tagline:** More Than a Mortgage. A Mortgage Solution.
- **Production domain:** https://www.theherbertteam.com
- **Primary conversions:** apply through the Primis HomeHub portal, or book a free 20-minute strategy call

Built with Astro 4 (static output), Tailwind CSS 3, and TypeScript in strict mode. There is no UI framework — the handful of interactive pieces (mobile nav, accordions, mortgage calculator, FAQ search, review filters) are small vanilla scripts, so the site ships essentially zero client JavaScript by default.

### Two rules worth keeping

1. **Never hardcode a client identifier.** Phone, email, NMLS numbers, apply URL, booking URL, form IDs, and analytics IDs all live in `src/data/site.ts`. Import from there.
2. **Never import images from `src/`.** Every image lives in `public/images/` and is referenced by absolute path with explicit `width`/`height` so there is no layout shift.

## How to add a blog post

Create a `.mdx` file in `src/content/blog/`. The filename becomes the URL slug (`my-post.mdx` → `/blog/my-post`). Frontmatter must satisfy the schema in `src/content/config.ts`:

```mdx
---
title: 'Why Your DTI Matters More Than Your Credit Score'
description: 'A 150-character summary that also serves as the meta description for search results and social shares.'
pubDate: 2026-03-14
category: 'Buying'
author: 'Jason Herbert'
image: '/images/blog/dti-explained.jpg'
imageAlt: 'A borrower reviewing a debt-to-income worksheet at a kitchen table'
featured: false
---

Body copy starts here. Use `##` for section headings — `#` is reserved for the
page title, which the layout renders.
```

- `category` must be one of the values in the schema: `Buying`, `Refinancing`, `VA`, `Self-Employed`, `Investing`, `Austin Market`, `Rates`. Categories drive the filter chips on `/blog`.
- `featured: true` promotes the post to the large card at the top of `/blog`. Keep exactly one post featured.
- Read time is calculated automatically from word count.
- Posts sort newest-first by `pubDate`. A future `pubDate` still builds, so set it deliberately.
- Article and BreadcrumbList JSON-LD are emitted automatically.

## How to replace images

1. Match the existing filename and path exactly — for example overwrite `public/images/hero/home-hero.jpg`.
2. Match the aspect ratio the placeholder uses, since `width`/`height` are hardcoded in the templates. The generator script lists every path with its intended dimensions; that file is the reference.
3. Compress before committing. Target under 300 KB for hero images and under 120 KB for cards. Hero images load eagerly with `fetchpriority="high"`, so their weight is directly on Largest Contentful Paint.
4. If the subject of the photo changes, update the `alt` text where it is set — in the page frontmatter, or in `programs.ts` / `solutions.ts` / `resources.ts` for data-driven pages.
5. `og-default.jpg` must stay 1200×630. Social platforms crop anything else badly.

Running `npm run placeholders` regenerates all placeholders and **will overwrite real photography**. Once real images are in, either stop running it or remove entries from the `images` array in `scripts/generate-placeholders.mjs`.

## Deployment

Static output — any host works. Build command `npm run build`, output directory `dist`, Node 18 or newer.

**Netlify** — connect the repo; the build command and publish directory above are the only settings needed. Add a `public/_redirects` file if you ever need redirects; Netlify serves `404.html` automatically.

**Vercel** — the Astro preset is detected automatically. No configuration required.

**Cloudflare Pages** — framework preset "Astro", build command `npm run build`, output directory `dist`.

### After the first deploy

- Point `www.theherbertteam.com` at the host and confirm the apex domain redirects to `www`, since `site` in `astro.config.mjs` is the `www` host and canonical URLs are built from it.
- Submit `https://www.theherbertteam.com/sitemap-index.xml` in Google Search Console.
- Verify the Meta Pixel fires with the Meta Pixel Helper extension and that GA4 realtime shows traffic.
- Test one live submission through each Pivot Point form and confirm it lands in the CRM with the right routing.
- Run Lighthouse on `/`, `/loan-programs/self-employed`, and `/resources/calculator`.
